## Уровни архитектуры

### 1. Perception Layer (Восприятие)

**Местоположение:** `modules/perception/`

**Назначение:** Нормализация и анализ сырого ввода от STT и сенсоров.

**Модули:**
- `input_normalizer.py` — Очистка и нормализация текста
- `emotion_detector.py` — Определение эмоционального состояния по тексту и голосу
- `intent_detector.py` — Идентификация намерений пользователя
- `__init__.py` — Единый интерфейс PerceptionLayer

**Входные данные:**
- Сырой текст от STT
- Характеристики голоса (опционально)
- Длительность тишины
- Флаг прерывания

**Выходные данные:**
```python
{
    "perceived_input": {
        "text": "Привет, как дела?",
        "speaker": "user",
        "timestamp": "2024-01-01T12:00:00",
        "context_signals": {
            "silence_duration": 5.0,
            "interruption": False
        }
    },
    "emotion": {
        "valence": 0.5,       # от -1.0 (негатив) до +1.0 (позитив)
        "arousal": 0.6,       # от 0.0 (спокойствие) до 1.0 (возбуждение)
        "dominant_emotion": "joy",
        "joy": 0.7,
        "sadness": 0.1,
        "anger": 0.0,
        "fear": 0.0,
        "surprise": 0.1,
        "neutral": 0.2
    },
    "intent": {
        "intent": "chat",
        "confidence": 0.85,
        "entities": {}
    }
}
```

**Поддерживаемые эмоции:**
- Радость (joy)
- Грусть (sadness)
- Гнев (anger)
- Страх (fear)
- Удивление (surprise)
- Отвращение (disgust)
- Нейтрально (neutral)

**Поддерживаемые намерения:**
- `query` — запрос информации
- `request` — просьба о действии
- `chat` — социальное общение
- `opinion` — запрос мнения
- `creative` — творческий запрос
- `problem` — сообщение о проблеме
- `confirmation` — подтверждение
- `recall` — воспоминание
- `filler` — слова-паразиты
- `unknown` — неизвестное

### 2. Interpretation Layer (Понимание)

**Местоположение:** `modules/interpretation/`

**Назначение:** Извлечение смысла, целей и важности из воспринятого ввода.

**Модули:**
- `topic_detector.py` — Определение тем и генерация эмбеддингов
- `goal_extractor.py` — Извлечение целей пользователя
- `importance_scorer.py` — Расчёт важности сообщения

**Выходные данные:**
```python
{
    "intent": "query",
    "topic_vector": [0.8, 0.2, 0.3, 0.7, 0.4],
    "topic": {
        "primary_topic": "ai_ml",
        "category": "technology",
        "keywords": ["ИИ", "нейросеть"],
        "related_topics": ["programming", "software"]
    },
    "user_emotion": 0.3,
    "emotion_full": {...},
    "goal": {
        "goal": "понять архитектуру ИИ",
        "goal_type": "informational",
        "priority": 0.7,    # 0.0 - 1.0
        "urgency": 0.5,     # 0.0 - 1.0
        "subgoals": []
    },
    "importance": 0.75,
    "importance_factors": {
        "emotion_intensity": 0.6,
        "urgency": 0.5,
        "goal_priority": 0.7,
        "topic_relevance": 0.8
    }
}
```

**Категории тем:**
- technology (программирование, ИИ, софт)
- science (физика, химия, биология, математика)
- entertainment (фильмы, музыка, игры, книги)
- lifestyle (здоровье, спорт, еда, путешествия)
- work (работа, бизнес, проекты)
- education (учеба, курсы, навыки)
- news (политика, экономика, общество)
- philosophy (смысл, ценности, религия)
- emotions (чувства, ментальное здоровье)

**Типы целей:**
- `informational` — получение информации
- `task` — выполнение задачи
- `creative` — творчество
- `social` — общение
- `emotional` — эмоциональная поддержка
- `decision` — помощь в решении

### 3. Memory Layer (Память)

**Местоположение:** `modules/memory/`

**Назначение:** Хранение и извлечение информации в нескольких системах памяти.

#### Уровни памяти:

| Тип | Хранилище | Назначение |
|-----|-----------|------------|
| **Эпизодическая** | SQLite/PostgreSQL | События, разговоры, контекст |
| **Семантическая** | Векторная БД | Знания, концепты, факты |
| **Реляционная** | Графовая БД | Связи между сущностями |
| **Личностная** | SQLite | Черты, ценности, состояние |

**Модули:**
- `episodic_memory.py` — Хранение событий и опыта
- `semantic_memory.py` — Хранение знаний и концептов
- `relational_memory.py` — Граф связей между сущностями
- `personality_memory.py` — Состояние личности
- `__init__.py` — Единый интерфейс MemoryLayer

**Пример эпизодической памяти:**

### 4. Personality Engine (Движок личности)

**Местоположение:** `modules/personality/`

**Назначение:** Расчёт динамической личностной позиции для каждой ситуации.

**Возможности:**
- Big Five traits (OCEAN) + дополнительные черты
- Система ценностей с расчётом выравнивания
- Отслеживание настроения
- Управление отношениями
- Обнаружение когнитивных конфликтов

**Черты личности (10):**
1. **Openness** — Открытость опыту
2. **Conscientiousness** — Добросовестность
3. **Extraversion** — Экстраверсия
4. **Agreeableness** — Доброжелательность
5. **Neuroticism** — Невротизм (эмоциональная стабильность)
6. **Curiosity** — Любопытство
7. **Creativity** — Креативность
8. **Empathy** — Эмпатия
9. **Humor** — Чувство юмора
10. **Assertiveness** — Уверенность

**Ценности:**
- Честность (honesty)
- Креативность (creativity)
- Свобода (freedom)
- Полезность (helpfulness)
- Знание (knowledge)
- Веселье (fun)
- Эффективность (efficiency)
- Традиции (tradition)

**Выходные данные (Stance):**
```python
PersonalityStance(
    topic_valence=0.3,        # Отношение к теме (-1 до +1)
    confidence=0.7,           # Уверенность (0 до 1)
    engagement_level=0.8,     # Уровень вовлечённости
    dominant_trait="curiosity",
    emotion_tone=0.4,         # Эмоциональный тон
    user_relationship=0.7,    # Отношения с пользователем
    value_alignment={
        "honesty": 0.8,
        "creativity": 0.6
    },
    cognitive_conflicts=[],   # Внутренние конфликты
    verbosity=0.6,            # Многословность
    initiative=0.7,           # Инициативность
    formality=0.3,            # Формальность
    emotional_expression=0.6  # Эмоциональная выразительность
)
```

### 5. Decision Layer (Уровень решений)

**Местоположение:** `modules/decision/`

**Назначение:** Выбор стратегии ответа на основе личности и контекста.

**Стратегии ответа:**
- `ANSWER_DIRECT` — Прямой ответ
- `ANSWER_DETAILED` — Развёрнутое объяснение
- `ASK_CLARIFYING` — Уточняющий вопрос
- `DECLINE` — Вежливый отказ
- `DEFLECT` — Переход к другой теме
- `ENGAGE_SOCIAL` — Социальное взаимодействие
- `PROVIDE_SUPPORT` — Эмоциональная поддержка
- `BRAINSTORM` — Генерация идей
- `COMPARE_OPTIONS` — Сравнение вариантов
- `BRIEF_ACK` — Краткое подтверждение

**Эмоциональные тона:**
- `NEUTRAL` — Нейтральный
- `WARM` — Тёплый
- `ENTHUSIASTIC` — Восторженный
- `EMPATHETIC` — Эмпатичный
- `PROFESSIONAL` — Профессиональный
- `PLAYFUL` — Игривый
- `SERIOUS` — Серьёзный

**Выходные данные (DecisionVector):**
```python
DecisionVector(
    strategy=ResponseStrategy.ANSWER_DETAILED,
    emotional_tone=EmotionalTone.WARM,
    verbosity=0.7,      # 0.0 (кратко) до 1.0 (подробно)
    initiative=0.6,     # 0.0 (реактивно) до 1.0 (проактивно)
    formality=0.3,      # 0.0 (неформально) до 1.0 (формально)
    include_examples=True,
    include_questions=True,
    include_personal_opinion=False,
    include_empathy=True,
    stay_on_topic=True,
    use_humor=False,
    max_length=400
)
```

### 6. Prompt Construction Layer (Конструктор промптов)

**Местоположение:** `modules/prompt_builder/`

**Назначение:** Построение контекстных промптов для LLM на основе всех предыдущих уровней.

**Компоненты промпта:**
1. Системный промпт с состоянием личности
2. Контекстный промпт из памяти
3. Инструкции по стилю из Decision Layer
4. Ограничения и рекомендации

**Пример системного промпта:**
```
Ты — Акари, ИИ-ассистент с динамической эволюционирующей личностью.

Текущее состояние личности:
- Доминирующая черта: любопытство
- Отношение к теме: +0.30
- Отношения с пользователем: 0.70
- Выравнивание ценностей: честность > креативность > свобода
- Уровень вовлечённости: 0.80

Выражай свою личность естественно через ответы.
```

### 7. LLM (Внешний уровень)

**Назначение:** Генерация текста на основе готового промпта.

**Интеграция:** Ollama (или другие LLM-провайдеры)

**Важно:** LLM не принимает решений о личности, а только выражает их через текст.

---

### 8. Behavior & Expression Layer (Поведение)

**Местоположение:** `modules/behavior/`

**Назначение:** Пост-обработка вывода LLM для выравнивания с личностью.

**Функции:**
- Применение речевых паттернов по чертам
- Корректировка тона и формальности
- Обеспечение эмпатии при необходимости
- Добавление уточняющих вопросов
- Применение юмора
- Очистка артефактов

**Речевые маркеры по чертам:**
- **Любопытство:** "Интересно...", "А что ты думаешь?"
- **Эмпатия:** "Понимаю...", "Я тебя слышу..."
- **Юмор:** "ха-ха", "представляешь..."
- **Добросовестность:** "Давайте разберемся...", "Важно отметить..."
- **Креативность:** "Представь себе...", "А что если..."
- **Экстраверсия:** "О!", "Кстати!", "Слушай!"

### 9. Evolution Layer (Эволюция / RL)

**Местоположение:** `modules/evolution/`

**Назначение:** Обучение с подкреплением для эволюции личности.

**Источники вознаграждения:**

| Источник | Вес | Описание |
|----------|-----|----------|
| Явная обратная связь | 1.0 | "Спасибо", "Плохо" |
| Неявная (эмоции) | 0.6 | Изменение эмоций пользователя |
| Достижение цели | 0.5 | Успешное выполнение задачи |
| Когнитивная согласованность | 0.3 | Внутренняя консистентность |
| Социальная связь | 0.4 | Длительность разговора |

**Формула вознаграждения:**
```python
reward = weighted_sum(
    user_explicit * 1.0,
    user_implicit * 0.6,
    goal_achievement * 0.5,
    cognitive_consistency * 0.3,
    social_bonding * 0.4
)
```

**Механизм эволюции:**
- Накопление истории вознаграждений (минимум 10)
- При превышении порога (0.3) — обновление черт
- Положительное вознаграждение → рост уверенности, любопытства
- Отрицательное → рост осторожности, невротизма

**Параметры эволюции:**
- `EVOLUTION_RATE = 0.05` — Скорость эволюции
- `EVOLUTION_THRESHOLD = 0.3` — Порог для изменений
- `MIN_REWARD_HISTORY = 10` — Минимум наград перед эволюцией

## Пример потока данных

```
Пользователь: "Привет! Как дела? Расскажи что-нибудь интересное про ИИ"

1. ВОСПРИЯТИЕ
   - Эмоция: радость (0.6), валентность: 0.5
   - Намерение: chat + query
   - Нормализованный текст: "Привет! Как дела? Расскажи про ИИ"

2. ИНТЕРПРЕТАЦИЯ
   - Тема: ai_ml (технологии)
   - Цель: узнать об ИИ (informational)
   - Важность: 0.65

3. ПАМЯТЬ
   - Сохранение эпизодической памяти
   - Извлечение недавних разговоров про ИИ
   - Получение состояния личности

4. ЛИЧНОСТЬ
   - Доминирующая черта: любопытство (0.7)
   - Валентность темы: +0.3 (любит тему ИИ)
   - Позиция: вовлечённый, энтузиастичный

5. РЕШЕНИЯ
   - Стратегия: ENGAGE_SOCIAL + ANSWER_DETAILED
   - Тон: ENTHUSIASTIC
   - Включить примеры: True

6. ПРОМПТЫ
   - Построение системного промпта с любопытством
   - Добавление контекста из памяти
   - Установка инструкций по стилю

7. LLM генерирует ответ

8. ПОВЕДЕНИЕ
   - Добавление маркеров энтузиазма
   - Обеспечение вовлекающего тона
   - Добавление уточняющего вопроса

9. Вывод пользователю

10. ЭВОЛЮЦИЯ (после реакции пользователя)
    - Расчёт вознаграждения от реакции
    - Обновление черт при достижении порога
```

## Пример использования

```python
from modules.cognitive_pipeline import CognitivePipeline

# Инициализация пайплайна
pipeline = CognitivePipeline(
    session_id="session_1",
    user_id="user_1",
    assistant_name="Акари"
)

# Обработка ввода пользователя
result = await pipeline.process(
    text="Привет! Расскажи про ИИ",
    voice_features=None,
    silence_duration=5.0
)

# Генерация ответа LLM (внешне)
llm_response = await generate_from_llm(
    prompt=result["prompt"],
    system=result["system_prompt"]
)

# Обработка ответа LLM
output = pipeline.process_llm_response(
    llm_response=llm_response,
    pipeline_result=result
)

# Отправка пользователю
print(output["text"])

# Позже: обработка обратной связи
feedback_result = pipeline.process_user_feedback(
    feedback="Спасибо, очень интересно!",
    pipeline_result=result
)
```
