# 🔍 Memory Inspector - Инструмент просмотра памяти

## 📋 Описание

Отдельный скрипт для просмотра и анализа всех типов памяти эволюционирующего ИИ-ассистента.

**Не требует** запуска основного `main.py` - работает напрямую с базами данных.

---

## 🚀 Быстрый старт

### Показать статистику

```bash
cd c:\AI
venv\Scripts\activate
python memory_inspector.py --stats
```

### Интерактивный режим

```bash
python memory_inspector.py
```

---

## 📖 Команды интерактивного режима

| Команда | Описание | Пример |
|---------|----------|--------|
| `stats` | Общая статистика всех типов памяти | `stats` |
| `eps [N]` | Эпизодическая память (последние N записей) | `eps 20` |
| `sem [N]` | Семантическая память (N концептов) | `sem 15` |
| `rel [name]` | Реляционная память (связи сущности) | `rel ИИ` |
| `per` | Личность (черты, ценности, настроение) | `per` |
| `search QUERY` | Поиск по всем типам памяти | `search программирование` |
| `export [file]` | Экспорт в JSON | `export backup.json` |
| `help` | Справка по командам | `help` |
| `quit` | Выход | `quit` |

## 🔧 Использование в Python коде

```python
from memory_inspector import MemoryInspector

# Создание инспектора
inspector = MemoryInspector()

# Показать статистику
inspector.show_statistics()

# Показать эпизодическую память
inspector.show_episodic(limit=20)

# Показать личность
inspector.show_personality()

# Поиск
inspector.search_memory("программирование", search_type="all")

# Экспорт
inspector.export_memory("backup.json")
```

## 🛠 Отладка

### Проблема: "Database locked"

**Решение:** Закройте `main.py` перед использованием инспектора.

```bash
# Остановить main.py (Ctrl+C)
# Затем запустить инспектор
python memory_inspector.py
```

### Проблема: "No memories found"

**Причина:** Базы данных пустые или находятся в другой папке.

**Решение:**
```bash
# Проверить наличие баз данных
dir data\*.db

# Если нет - запустить main.py для создания
python main.py
```

### Проблема: Unicode символы не отображаются

**Решение для Windows:**
```bash
chcp 65001
python memory_inspector.py
```

## 📊 Типы памяти

### 1. Эпизодическая (`episodic_memory.db`)

**Хранит:**
- Разговоры (user/assistant messages)
- Эмоциональную оценку
- Важность событий
- Темы и намерения

**Команда:** `eps [N]`

### 2. Семантическая (`semantic_memory.db`)

**Хранит:**
- Концепты и знания
- Векторные эмбеддинги
- Категории и теги

**Команда:** `sem [N]`

### 3. Реляционная (`relational_memory.db`)

**Хранит:**
- Сущности (entities)
- Связи между сущностями (relations)
- Силу связей

**Команда:** `rel [name]`

### 4. Личностная (`personality_memory.db`)

**Хранит:**
- 10 черт личности
- 8 ценностей с весами
- Текущее настроение
- Отношения с пользователями

**Команда:** `per`

## 💡 Советы

1. **Регулярный экспорт:**
   ```bash
   memory> export backup_$(date).json
   ```

2. **Поиск проблем:**
   - Проверьте эмоции: `eps 10` (смотрите на valence)
   - Проверьте личность: `per` (смотрите на mood)
   - Поиск конфликтов: `search конфликт`

3. **Анализ эволюции:**
   - Экспортируйте память до и после сессии
   - Сравните `personality` секции

4. **Отладка LLM:**
   - Проверьте что сохранилось: `eps 5`
   - Проверьте контекст: `search тема`
